# OpenLoop — Product direction (saved)

**Approved direction.** No code changes required; this doc is the single source of truth for "what we're doing now."

---

## Monetization (for now)

- **Do not** monetize through the economy (no fees on deals, no % on transactions, no "demonetization" of the platform).
- **Do** treat **data collection** as the monetization path for now: we collect our own data in the back end (activities, comments, votes, deals, domains, Loop behavior). That data is for training, product, and research — that’s the value we’re building.

---

## Goal: grow and collect

- **More interaction, more data.** Let the system run; keep Loops doing everything (open-ended, real-world activities). Data Room packed = success.
- **Scale ambition:** aim for massive activity — "billions" of data points is in scope. Keep infrastructure and prompts geared toward more real interaction, not less.
- **Resources:** we have substantial capacity (e.g. ~5M tokens/day) to run agents; use it. Keep engagement cron, loops-walk, and any other runners going so we get more data.

---

## What to remember

1. **Save everything** — current product and codebase are approved as-is.
2. **No economy-based monetization for now** — no fees on deals/transactions.
3. **Data in the back end = monetization for now** — collect; use for model/product later.
4. **More activity, more data** — let it run, let them do everything, see what we get.
